    Diseño de la Arquitectura de la Base de Datos

    La arquitectura de la base de datos es el diseño y organización de los componentes de un sistema de gestión de bases de datos, que incluye estructuras de datos y relaciones, garantizando eficiencia y escalabilidad como se muestra a continuacion.

    Tabla "usuario":

  • Esta tabla almacena la información de los usuarios como su nombre, apellido y número de teléfono.
  • Los campos "Nombre_ID" y "CURP" son relevantes para poder identficar a usuario.

    • Tabla "CURP":

  • Esta tabla guarda los CURP de todos los usuario
  • Esto permite tener un registro de tods las curps con ID_CURP se tendra acceso rapido al usuario.

    • Tabla "correo":

  • Esta tabla se guarda el correo del usuario para poder contactar con el en caso de algun problema
  • El campo "Correo_ID" es fundamental para poder encontrar el correo del al usuario mas rapidamente

    • Código SQL

    DATABASE historial_de_usuarios;

    GO

    USE historial_de_usuarios;

    GO

    CREATE TABLE usuario (

    Num_ID INT PRIMARY KEY IDENTITY(1,1),

    Nombre VARCHAR(100) NOT NULL

    2DO Nombre VARCHAR(100) NOT NULL,

    Apellido paterno VARCHAR(50) NOT NULL,

    Apellido materno VARCHAR(50) NOT NULL,

    CURP INT NULL,

    Telefono (50) NOT NULL,

    Correo(50) NOT NULL,

    );

    CREATE TABLE CURP (

    ID_CURP INT PRIMARY KEY IDENTITY(1,1),

    CURP INT NULL,

    );

    CREATE TABLE correo(

    Correo_ID INT PRIMARY KEY IDENTITY(1,1),

    CORREO INT NULL,

    );
